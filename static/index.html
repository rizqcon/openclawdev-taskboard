Task Board Help

Overview

The Jarvis Task Board is a multi-agent project management system. Tasks flow through stages while AI agents collaborate to complete work under human supervision.

Workflow Stages

Backlog In Progress Review Done

Blocked can branch from In Progress when waiting on info

  • Backlog: Tasks waiting to be started
  • In Progress: Agent actively working (auto-spawns AI session)
  • Review: Work complete, awaiting User's approval
  • Done: Only User can move tasks here (final approval)
  • Blocked: Waiting on external input or decision

Agents

Jarvis Main coordinator, handles command bar chat
Architect System design, patterns, scalability
Security Auditor SOC2, HIPAA, CIS compliance
Code Reviewer Code quality, best practices
UX Manager User flows, UI consistency
Alberto Human supervisor (you!)

Auto-Spawn: How Agents Activate

  1. Assign a task to an agent (e.g., Architect)
  2. Drag the card to In Progress
  3. The agent's AI session spawns automatically
  4. Agent posts updates as comments on the card
  5. Agent moves to Review when done
  6. You review and move to Done (or request changes)

Session Persistence

Each agent maintains a persistent session per task:

  • Sessions stay alive while task is In Progress or Review
  • You can have back-and-forth conversations via comments
  • Context is preserved no cold starts for follow-ups
  • Sessions terminate when task moves to Done

Multi-Agent Collaboration

To bring additional agents into a conversation:

  1. Open a card and go to the chat section
  2. Use the @mention button to tag agents
  3. Tagged agents receive a notification and can respond
  4. Each agent maintains their own session context

Example: Architect working on a feature can @Security Auditor for a quick review without creating a separate task.

Action Items

Cards can have action items (shown as notification badges):

  • Question Agent needs clarification
  • Completion Auto-created when moved to Review
  • Blocker Auto-created when moved to Blocked

Click the checkbox to resolve an action item, or click to delete it.

Command Bar

Click the search bar in the header to open a direct Chat with Jarvis:

  • Ask questions about the project
  • Give high-level instructions
  • Paste images or files (Ctrl+V)
  • Click to expand to full size

Visual Indicators

  • Glowing card AI agent actively working (consuming tokens)
  • Red badge Open action items needing attention
  • Agent colors Each agent has a distinct color tag
  • Priority colors Critical (red), High (orange), Medium (yellow), Low (green)

Security

  • All agent sessions run with guardrails (filesystem boundaries, forbidden actions)
  • Agents cannot access paths outside authorized directories
  • External API calls require approval via action items
  • Only User can finalize tasks (move to Done)
